Pretend Acme Corp

This page simulates a 3rd-party SaaS embedding the echoboard widget. Look bottom-right for the floating Feedback button.

1 · Guest mode (default)

If your visitors aren't signed into your product, the widget collects their email + name on first submit. No code needed — just the <script> tag at the bottom of this page.

2 · Auto-identify a signed-in user

If the visitor is already signed into your product, your frontend calls echoboard.identify({...}) with their identity — they skip the email gate entirely.

Open the widget after each click. The "Submitting as …" line in the Submit tab reflects whoever's currently identified.

3 · Auto-identify with HMAC (production-grade)

For production, your backend mints an HMAC-signed token (using the identifySecretKey from your echoboard dashboard) and your frontend passes it to echoboard.identify({ token }). This makes browser-side impersonation impossible.

Code samples for Node / Python / Ruby / PHP are in /dashboard/settings/widget. Toggle "Require signed identity tokens" on the same page to reject unsigned identify calls workspace-wide.

The install snippet

Exactly what we ship customers in the dashboard:

<script src="/widget.js" data-board-id="..." async></script>
